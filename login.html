Component

Login

Identify yourself by inputting email address and password to gain an access to the system.

Overview

When the user clicks on Sign In button in header, a login box will appear in the middle of the screen. After the user fills out the email address and password, then clicks on Submit button, the application will do client side validation. If the validation succeeds, Login API will be called.

How to Achieve this Feature/Technical

To achieve all features available in Login, API integration is needed to interact between application with database. API is responsible for handling request by user who have entered their email address and password. Login API returns response in JSON format and current user's session id will be saved to the session storage for the next API request purpose. API Integration

Following is service of 'Login'

API Name : Login API

Request Header : clientID: manulife; tokenID: MANULIFEWEBAPP

Service URL : POST http://manulife-claim-dockermgmt.centralus.cloudapp.azure.com:6060/user/login

Request Body

        
              {
                "email":"alexa.sawyer@gmail.com",
                "password":"ABC123abc"
              }

Response

        
                {
                  "status": "success",
                  "sessionId": "xsiood34567$232"
                }
